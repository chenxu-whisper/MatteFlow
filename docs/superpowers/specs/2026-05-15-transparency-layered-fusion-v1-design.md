# 半透明分层与软融合 V1 设计

**目标**

在不显著破坏当前实体主体抠像质量的前提下，提升绿幕与黑底素材中半透明内容的保留质量，重点改善辉光、粒子、发光边、薄烟等区域的 alpha 过渡自然度、边缘颜色一致性和视频时序稳定性。

**范围**

- 覆盖 `GREEN_SCREEN` 与 `BLACK_BACKGROUND` 两条链路。
- 第一版尽量不新增 GUI 控件，优先复用现有 `transparency_preserve`、`green_despill_strength`、`edge_despill_factor`、黑底相关参数。
- 只改与半透明融合、半透明颜色修复、半透明时序稳定直接相关的逻辑。
- 不在本次范围内引入新的大模型或外部依赖。

**非目标**

- 不重做整个传统绿幕算法。
- 不在本次引入视觉大模型或多模型路由。
- 不对 GUI 做大改版，只允许补充必要状态日志或内部调试输出。

## 现状问题

当前实现已经具备一定的半透明保留能力，但仍存在三个核心问题：

1. 实体主体与半透明特效仍主要共用一条 alpha 逻辑，导致主体保护与特效保留相互牵制。
2. 半透明层的融合偏硬，主要依赖 `np.maximum()` 与固定幂函数，弱辉光和细粒子容易被压缩，灰阶过渡不够自然。
3. 颜色修复已考虑半透明区，但仍以“边缘去绿/去黑”为主，缺少针对半透明层的 alpha-aware RGB 修复策略，导致合成后出现发灰、发脏、暗边或色偏。

## 方案选择

### 方案 A：继续增强现有阈值规则

- 优点：改动小，回归风险低。
- 缺点：仍然是一条 alpha 兼顾实体与半透明，质量上限有限。

### 方案 B：分层 + 软融合 + alpha-aware RGB 修复

- 优点：在当前架构基础上即可实现，能直接提升半透明观感与稳定性。
- 缺点：需要改动 `hybrid_matte` 与 `color_decontaminate` 的核心逻辑，并补充新的回归测试。

### 方案 C：引入额外 AI 模型做半透明评估

- 优点：潜在能力更强。
- 缺点：成本高、速度慢、验证复杂，不适合作为第一版。

**结论**

采用方案 B，作为 V1 正式实现方向。

## 总体设计

V1 将现有输出 alpha 逻辑显式拆分为两层：

- `solid_alpha`：用于实体主体、实体边缘、白色软主体等需要稳定实心保留的区域。
- `effect_alpha`：用于辉光、粒子、发光边、薄烟、弱半透明层。

最终结果不再使用简单 `np.maximum(solid_alpha, effect_alpha)` 作为唯一融合策略，而是改为更符合“实体之上叠加特效层”的软融合形式：

`final_alpha = solid_alpha + effect_alpha * (1 - solid_alpha)`

这一融合方式有两个目的：

- 当实体主体已经较实的时候，不会因为特效层再次硬抬 alpha，避免边缘发硬。
- 当实体主体较弱或不存在时，effect 层可以连续保留，形成更自然的灰阶过渡。

## 文件与职责

### `src/matteflow/matte/hybrid_matte.py`

职责调整：

- 保留现有的基础 matte 生成逻辑。
- 将现有绿幕特效保留逻辑重构为“实体层 + 特效层 + 软融合”。
- 为黑底素材引入对应的 effect alpha 估计与融合。
- 增加半透明层软曲线函数，替换当前固定 `power(2.0)` 风格的 effect 抬升方式。

### `src/matteflow/refine/color_decontaminate.py`

职责调整：

- 保留当前的绿边/黑边修复框架。
- 增加针对半透明区的 alpha-aware RGB 修复：
  - 绿幕：在去绿基础上保粉色/白色辉光结构，避免暗边和灰雾。
  - 黑底：在提亮基础上保护粒子色彩，避免统一增亮导致发灰。
- 将半透明层处理与实体主体修复区分开来，避免强修复污染实体主体。

### `src/matteflow/pipeline.py`

职责调整：

- 保持处理流程顺序基本不变。
- 在时序稳定阶段引入“只针对半透明层”的稳定策略接口。
- 不在 V1 中改动整体输出目录或编码格式。

### `src/matteflow/config.py`

职责调整：

- 尽量不增加新的 GUI 暴露参数。
- 如实现需要，可增加内部配置字段，但默认值应安全，并由现有参数驱动。
- 如果新增内部字段，不在 GUI 主面板暴露。

## 绿幕链路设计

### 1. 实体层 `solid_alpha`

绿幕实体层来源于当前 `base_alpha`，并继续使用：

- 白色主体保护
- 柔和浅色主体保护
- 高置信度前景保留

`solid_alpha` 的目标是稳定保住兔子、耳朵、本体、亮色主体边缘，不因为半透明特效增强而意外透明。

### 2. 特效层 `effect_alpha`

V1 的 `effect_alpha` 来源于当前基础 alpha 与颜色权重联合估计，但做三项改造：

- 使用更柔和的归一化策略，避免弱特效被 floor 直接压没。
- 用 `smoothstep` 风格的软曲线替代固定 `power(2.0)`。
- 保留颜色权重，但让它只作用于特效层，不直接干预实体层。

V1 重点保留以下区域：

- 粉色辉光
- 白色辉光
- 低饱和亮色薄边
- 非绿幕色的弱半透明边缘

### 3. 绿幕最终融合

绿幕最终 alpha 采用：

- `solid_alpha` 作为实体主体主导层
- `effect_alpha * transparency_preserve` 作为半透明特效层
- `final_alpha = solid_alpha + effect_alpha * (1 - solid_alpha)`

必要时再做一次轻量裁剪与后处理，确保 alpha 保持在 `[0, 1]` 且不引入额外硬边。

## 黑底链路设计

### 1. 黑底实体层

黑底素材的实体层继续依赖当前黑底 matte 主逻辑。

### 2. 黑底特效层

针对黑底新增 effect 层估计，重点保留：

- 暗背景上的亮粒子
- 弱发光边
- 薄烟和半透明亮雾

effect 层判断重点依赖：

- 亮度局部峰值
- alpha 中低区的连续结构
- 色彩范围与局部对比

### 3. 黑底最终融合

同样采用实体层与特效层的软融合，而不是单纯“暗部拉亮 + alpha 叠加”。

## alpha 软曲线设计

V1 引入一个统一的 effect 软曲线函数，目标如下：

- 弱信号不立刻被压扁
- 中间灰阶过渡更连续
- 强信号不会过早饱和到 1

推荐实现形式：

- 基于 `smoothstep(low, high, x)` 的单调 S 曲线
- 或“先轻提升再轻压缩”的分段曲线

V1 不引入复杂曲线编辑器，使用内部固定函数即可。

## alpha-aware RGB 修复设计

### 绿幕

对 `0.02 < alpha < 0.75` 的区域执行半透明修复：

- 去绿强度按 alpha 连续变化
- 保粉色和白色辉光的原始色相关系
- 对低亮度半透明边缘做定向补亮，避免黑边
- 避免对白色主体区做过强补色，防止变粉或变脏

### 黑底

对 `0.02 < alpha < 0.75` 的区域执行半透明修复：

- 对暗粒子做局部提亮而不是整体抬升
- 对灰化区域恢复部分饱和度
- 对已有强颜色的粒子保留原色比例，避免统一去灰造成脏色

## 半透明时序稳定

V1 只对视频素材启用半透明层专属稳定：

- 只平滑 `0.03 ~ 0.75` 范围的 alpha
- 不对高置信度实体主体做强时序平滑
- 目标是减少粒子、辉光、发光边闪烁

实现方式保持轻量：

- 复用现有时序稳定框架
- 增加“半透明区掩码”约束

## 日志与调试输出

V1 增强日志，但不增加新的 GUI 主控项：

- 输出实体层平均 alpha、特效层平均 alpha、融合后平均 alpha
- 记录 effect 层占比与半透明时序稳定是否启用
- 在 debug 模式下允许输出中间 effect alpha 统计信息

## 测试策略

### 单元测试

新增或扩展以下测试：

- 绿幕辉光在新融合策略下保留连续灰阶
- 绿幕白色软边不会因为 effect 层增强而整体实心化或透明化
- 黑底粒子在 effect 层策略下不应整体消失
- alpha 软曲线保持单调，且输出范围稳定在 `[0,1]`
- 半透明 RGB 修复不会对高 alpha 实体区产生明显副作用

### 回归测试

回归范围至少覆盖：

- `test_auto_params.py`
- `test_web_gui_defaults.py`
- `test_color_decontaminate.py`
- `test_logging_instrumentation.py`
- 新增半透明融合专项测试

### 手动验证素材

至少应包含：

- 绿幕辉光素材
- 绿幕白色主体 + 发光边素材
- 黑底粒子/烟雾素材
- 视频样本，验证是否闪烁

## 风险与控制

### 风险 1：主体边缘被过度柔化

控制措施：

- effect 层只叠加到 `1 - solid_alpha` 的剩余空间
- 高置信度实体层不受 effect 层直接覆盖

### 风险 2：半透明区保得太多，背景雾边回流

控制措施：

- effect 层必须经过颜色权重与软曲线共同约束
- 保留现有 `transparency_preserve` 作为总量控制

### 风险 3：黑底亮雾被过度提亮发灰

控制措施：

- 优先做局部对比恢复与色彩保护，不做全局统一提亮

### 风险 4：视频时序平滑拖影

控制措施：

- 只对半透明层做轻量平滑
- 不对主体实体区做强制平均

## 实施顺序

V1 分三步落地：

1. 先在 `hybrid_matte.py` 完成实体层/特效层/软融合重构。
2. 再在 `color_decontaminate.py` 加入 alpha-aware RGB 修复。
3. 最后补充半透明区时序稳定与测试回归。

## 成功标准

满足以下条件即视为 V1 达标：

- 绿幕辉光和粒子的 alpha 过渡较当前版本更连续
- 白色软边主体没有明显变空或发硬
- 黑底粒子与亮雾较当前版本更稳定、更少发灰
- 视频中半透明区域闪烁感下降
- 不引入主体整体质量明显退化
